# Unity Collective - Feature Documentation Index

This directory contains comprehensive documentation for each feature in the Unity Collective platform. Each document is designed to serve as a **primer for coding agents** who need to understand, modify, or extend a specific feature.

## Purpose

These feature documentation files provide:

1.  **What the feature does** - A clear description of the feature's purpose and functionality.
2.  **Files involved** - A complete list of all files that implement or support the feature.
3.  **How to make changes** - Detailed instructions on how to modify existing behavior.
4.  **How to add items** - Step-by-step guidance on extending the feature with new functionality or data.

## How to Use This Documentation

### **For Coding Agents**

When tasked with modifying or extending a feature:

1.  **Read the corresponding feature documentation** to understand what the feature does and how it is implemented.
2.  **Review the "Files Involved" section** to know which files you need to examine or modify.
3.  **Follow the "How to Make Changes" section** for guidance on modifying existing behavior.
4.  **Follow the "How to Add Items" section** for step-by-step instructions on extending the feature.

### **For Developers**

Use these documents as a quick reference when working on a specific feature. They provide a high-level overview and practical code examples to accelerate development.

## Document Structure

Each feature documentation file follows this consistent structure:

1.  **What it Does** - A description of the feature's purpose and functionality.
2.  **Files Involved** - A table listing all relevant files and their purposes.
3.  **How to Make Changes** - Instructions for modifying existing behavior, with code examples.
4.  **How to Add Items** - Instructions for extending the feature with new functionality or data, with code examples.

## Contributing

If you add a new feature to the Unity Collective platform, please create a corresponding feature documentation file following the structure outlined above. This ensures that future developers and coding agents can quickly understand and work with your feature.
